## 1. Fundamentos Teóricos

### 1.1 Spreading Activation Theory

Priming semântico é um fenômeno cognitivo onde a exposição a um estímulo (prime) facilita o processamento de estímulos relacionados (target). Em redes neurais biológicas, isso ocorre via **spreading activation**: energia que se propaga de um nó ativado para seus vizinhos através das sinapses.

**Exemplo:**
```
Prime: "Hospital"
↓ (ativação espalha)
Targets pré-ativados: "médico" (0.85), "remédio" (0.72), "exame" (0.65)
```

### 1.2 Implementação em Grafo

EVA-Mind mapeia conceitos semânticos para nós em Neo4j, onde:
- **Nós** = Conceitos (Eventos, Tópicos, Significantes, Pessoas)
- **Arestas** = Relações semânticas (RELATED_TO, EVOCA, MENTIONED)
- **Peso** = Força da conexão (co-ocorrência histórica)

```cypher
(:Event {content: "Fui ao hospital"})-[:RELATED_TO]->(:Topic {name: "Saúde"})
                                     -[:EVOCA]->(:Significante {text: "Médico"})
```

### 2.2 Algoritmo de Spreading Activation

**Implementação (`fdpn_engine.go`):**

```go
// Pseudo-código simplificado
func (e *FDPNEngine) primeKeyword(ctx, userID, keyword string) error {
    // 1. Encontrar nó raiz
    query := `
        MATCH (raiz) 
        WHERE toLower(raiz.nome) CONTAINS toLower($keyword)
        WITH raiz LIMIT 1
        
        // 2. Spreading Activation (depth 3)
        MATCH path = (raiz)-[r*1..3]-(vizinho)
        
        // 3. Calcular energia com decay exponencial
        WITH raiz, vizinho, relationships(path) as rels,
             reduce(energy = 1.0, rel IN rels | energy * 0.85) as activation
        
        WHERE activation >= 0.3  // Threshold
        
        RETURN raiz, collect(vizinho) as activated_nodes
    `
    
    // 4. Cache em L1 (Memory) e L2 (Redis)
    subgraph := buildSubgraph(results)
    e.localCache.Store(cacheKey, subgraph)  // L1
    e.redis.Set(cacheKey, subgraph, 5*time.Minute)  // L2
}
```

**Parâmetros de Decay:**
- Depth 1: energia = 1.0 (vizinhos diretos)
- Depth 2: energia = 0.85 (2º grau)
- Depth 3: energia = 0.72 (3º grau)
- Threshold: 0.3 (descarta ativações fracas)

```go
func (s *Synaptogenesis) UpdateCoOccurrence(event *Event) {
    for i, entity1 := range event.Entities {
        for j := i+1; j < len(event.Entities); j++ {
            entity2 := event.Entities[j]
            
            // Criar/reforçar sinapse

                ON CREATE SET r.weight = 1, r.created = datetime()
                ON MATCH SET r.weight = r.weight + 1, r.last_activation = datetime()
            `
        }
    }
}
```

## 5. Filtragem de Entropia (Absolute Zero)

### 5.1 Problema do Ruído Semântico

Spreading activation pode gerar **falsos positivos**:

```
"Hospital" → médico (0.85) ✓
         → remédio (0.72) ✓
         → pizza (0.35) ✗ (ruído: co-ocorreu 1x por acaso)
```

### 5.2 Solução: Dynamic Thresholding

```go
func (e *FDPNEngine) filterEntropy(nodes []ActivatedNode) []ActivatedNode {
    // Encontrar ativação máxima
    maxAct := 0.0
    for _, n := range nodes {
        if n.Activation > maxAct {
            maxAct = n.Activation
        }
    }
    
    // Manter apenas >= 20% do máximo
    filtered := []ActivatedNode{}
    for _, n := range nodes {
        if n.Activation >= maxAct * 0.2 {
            filtered = append(filtered, n)
        }
    }
    return filtered
}
```

**Resultado:**
- "Hospital" → médico (0.85) ✓
- "Hospital" → remédio (0.72) ✓
- "Hospital" → pizza (0.35) ✗ (0.35 < 0.85*0.2 = 0.17... wait, isso passa!)

Ajuste dinâmico: se `maxAct * 0.2` ainda deixa passar ruído, usar threshold absoluto de 0.3 como fallback.

```go
type FDPNEngine struct {
    activeThreads chan struct{} // Semáforo: max 10 threads
}

func (e *FDPNEngine) StreamingPrime(keywords []string) {
    var wg sync.WaitGroup
    for _, kw := range keywords {
        wg.Add(1)
        e.activeThreads <- struct{}{} // Acquire
        
        go func(keyword string) {
            defer wg.Done()
            defer func() { <-e.activeThreads }() // Release
            
            e.primeKeyword(ctx, userID, keyword)
        }(kw)
    }
    wg.Wait()
}
```
